 Выполненные этапы разработки (Блок A)
Проект был реализован поэтапно: от структуры данных до сложной логики взаимодействия с пользователем.

1. Основы и Профиль (Task-1)
HTML5 & CSS3: Создал семантическую структуру карточки пользователя.

Flexbox: Использовал display: flex для выравнивания контента и создания сетки профиля.

Стилизация: Настроил визуальные эффекты (тени, скругления, типографика).

2. Интерактивная навигация (Гамбургер-меню)
JS Logic: Реализовал «оживление» меню.

Использовал document.querySelector для поиска иконки меню и списка ссылок.

Настроил addEventListener('click', ...) для отслеживания нажатий.

Применил classList.toggle('active'), чтобы плавно показывать и скрывать меню на мобильных устройствах.

CSS Animations: Добавил плавность переходов через transition, чтобы меню не просто «прыгало», а выезжало.

3. Функционал авторизации (Логин и Вход)
UI/UX: Сверстал форму входа с полями для ввода данных (Email/Password).

Валидация: Реализовал проверку вводимых данных в index.js перед отправкой.

Обработка данных: Настроил перехват события submit, чтобы имитировать процесс входа в систему без перезагрузки страницы.

🧠 Анализ проделанной работы
Что было сложно:
Связка Гамбургер-меню и Логина: Было непросто сделать так, чтобы форма входа корректно отображалась поверх основного контента, особенно на маленьких экранах. Пришлось глубоко разбираться в z-index и позиционировании fixed/absolute.

Логика переключения состояний: Настройка корректного открытия и закрытия меню (чтобы оно закрывалось при клике на ссылку или вне области меню) потребовала написания дополнительных условий в JavaScript.

Адаптивность форм: Сделать форму логина удобной для ввода данных с телефона (крупные поля, правильные отступы) оказалось сложнее, чем просто сверстать её для ПК.

Что понравилось:
Работа с логикой (JS): Больше всего затянуло создание функционала входа. Было круто видеть, как код проверяет, что ввел пользователь, и выдает соответствующую реакцию.

Результат «Гамбургера»: Очень понравилось, как чисто сработал переход от десктопного меню к мобильному «бургеру» — это придало проекту вид настоящего профессионального приложения.

Чистота кода: Я старался разделять стили и скрипты по отдельным файлам, что помогло мне не запутаться, когда проект начал расти от простой карточки до системы с авторизацией.
